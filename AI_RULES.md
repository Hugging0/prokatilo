# 🧠 ПРОКАТило — Мастер-контекст и Правила ИИ (AI_RULES.md)

## 1. Контекст проекта
- **Название:** ПРОКАТило.
- **Суть:** MVP сервиса аренды вещей и техники. 
- **Ключевое УТП:** Доставка за 15 минут, аренда без залога.
- **Бизнес-модель:** B2C (вещи сдаются централизованно владельцем сервиса, это НЕ шеринг между соседями).
- **Стартовый инвентарь:** PlayStation, PlayStation VR, робот-мойщик окон, моющий пылесос для бытовой мебели.
- **Архитектура:** Раздельный Fullstack (Next.js на фронтенде, FastAPI на бэкенде).

## 2. Технологический стек
- **Frontend:** Next.js (App Router), React, Tailwind CSS, TypeScript, Zustand (стейт), Lucide React (иконки).
- **Backend:** FastAPI, Python, SQLAlchemy, Pydantic.

## 3. Железные правила написания кода (НЕ НАРУШАТЬ)
1. **Строгий TypeScript:** Никаких `any`. Все интерфейсы (`Item`, `Order`, `User`) должны быть строго описаны в `src/types`.
2. **Язык интерфейса:** Весь UI, плейсхолдеры и уведомления — СТРОГО на русском языке.
3. **Модульность (FSD-подобная архитектура):**
   - Страницы лежат только в `src/app`.
   - Переиспользуемые кнопки, инпуты — в `src/components/ui`.
   - Сложные блоки (список заказов, форма) — в `src/components/features`.
   - Утилиты и константы — в `src/lib`.
4. **Запрет на спагетти-код:** Компоненты не должны превышать 150-200 строк. Если файл растет — декомпозируем.
5. **Чистый импорт:** Использовать алиасы `@/components/...` вместо относительных путей `../../`.

## 4. Правила рабочего процесса (Workflow)
- Каждый логический шаг завершается коммитом.
- Использовать конвенцию коммитов (feat: ..., fix: ..., chore: ..., refactor: ...).
- При добавлении новых сущностей (например, новой таблицы в БД или нового глобального стейта) этот файл ДОЛЖЕН БЫТЬ ОБНОВЛЕН.
